# 协作与提交规范

## 推荐流程

1. 从 `main` 创建功能分支，例如 `feature/member-delete`。
2. 每次提交只处理一个明确主题，提交前检查 `.ui`、C++ 和 SQL 是否同步。
3. 推送分支并创建 Pull Request，由另一位成员检查后合并。
4. 在报告中使用 GitHub 的 Commits、Branches 或 Pull Requests 页面截图作为协作证据。

## 提交信息

- `feat:` 新功能
- `fix:` 缺陷修复
- `docs:` 文档更新
- `test:` 测试与测试数据
- `refactor:` 不改变功能的代码整理

示例：`feat: 增加商品和会员安全删除功能`

## 提交前检查

- Qt 工程可以重新构建。
- SQL 初始化脚本和增量迁移脚本都已更新。
- 新增、修改、删除操作包含输入校验和用户提示。
- README 的运行步骤和演示账号仍然准确。
- 不提交密码、个人令牌、IDE 用户配置和新的构建产物。
