# 超市销售管理系统

基于 **C++17、Qt 6 Widgets、Qt SQL 与 MySQL 8.0** 开发的桌面端超市销售管理系统，覆盖收银、库存、会员和经营报表四类业务。项目为软件工程课程设计，重点展示需求分析、分层设计、数据库约束、存储过程和基于 Git 的协作过程。

## 功能概览

| 模块 | 已实现功能 |
| --- | --- |
| 用户与权限 | 员工账号登录、MD5 密码校验、收银员/库存管理员/经理角色控制 |
| 销售管理 | 商品搜索、购物车、会员折扣、四种支付方式、事务化结账、库存扣减 |
| 库存管理 | 商品新增/编辑/安全删除、进货、库存筛选、低库存预警 |
| 会员管理 | 会员新增/查询/编辑/安全删除、积分与折扣更新、消费统计 |
| 经营报表 | 按时间查询销售数据、商品销售排行、会员消费排行 |

“安全删除”不会破坏历史业务数据：已有销售/进货记录的商品、已有消费记录的会员会被数据库存储过程拒绝删除；无业务历史的记录可以正常删除。

## 技术架构

```text
Qt Widgets 界面层
        ↓
C++ 业务逻辑层（SalesWidget / inventory_widget / member_widget / report_widget）
        ↓
Qt SQL 数据访问层（DatabaseManager）
        ↓
MySQL 数据库层（表、视图、存储过程、触发器与权限）
```

## 环境要求

- Windows 10/11
- Qt 6.9.0，MinGW 64-bit 组件
- Qt MySQL 驱动 `QMYSQL`
- MySQL 8.0
- Qt Creator（推荐）

## 快速运行

1. 在 MySQL 8.0 中以管理员身份执行 [`课程设计.sql`](./课程设计.sql)。脚本会重建 `超市销售管理系统` 数据库并导入演示数据。
2. 确认 MySQL 监听 `localhost:3306`，初始管理员密码与 `main.cpp` 中的连接配置一致。当前示例配置为 `root/root`。
3. 使用 Qt Creator 打开 `SupermarketSystem.pro`，选择 **Desktop Qt 6.9.0 MinGW 64-bit** 套件。
4. 确保 Qt 的 MySQL 驱动可用后构建并运行。
5. 使用以下演示账号登录：

| 角色 | 用户名 | 密码 |
| --- | --- | --- |
| 经理 | `manager` | `manager123` |
| 收银员 | `cashier1` | `cashier123` |
| 库存管理员 | `stock1` | `stock123` |

如果数据库已经由旧版脚本创建，不需要清库；只需执行 [`database/migrations/20260621_add_delete_operations.sql`](./database/migrations/20260621_add_delete_operations.sql)，即可增加商品/会员安全删除存储过程及权限。

## 删除功能验收

### 商品删除

1. 使用库存管理员或经理账号进入“库存”模块。
2. 新增一个测试商品但不要进货或销售。
3. 选中该商品，点击“删除商品”，确认后记录从列表消失。
4. 尝试删除已有进货或销售记录的商品，系统应提示为保证历史数据完整性不能删除。

### 会员删除

1. 使用收银员或经理账号进入“会员”模块。
2. 新增一个测试会员但不要使用该会员结账。
3. 选中该会员，点击“删除会员”，确认后记录从列表消失。
4. 尝试删除已有消费记录的会员，系统应提示为保证历史数据完整性不能删除。

## 项目结构

```text
SupermarketSystem/
├─ main.cpp / mainwindow.*          # 程序入口与主窗口
├─ logindialog.*                    # 登录与角色连接
├─ saleswidget.*                    # 销售模块
├─ inventory_widget.*               # 库存模块
├─ member_widget.*                  # 会员模块
├─ report_widget.*                  # 报表模块
├─ databasemanager.*                # 数据库连接单例
├─ 课程设计.sql                      # 完整数据库初始化脚本
└─ database/migrations/             # 已有数据库的增量升级脚本
```

## 已知限制

- 数据库连接参数目前写在 `main.cpp`，正式部署建议改为配置文件或环境变量。
- 商品和会员采用“有历史记录则禁止物理删除”的策略；生产系统可进一步升级为带状态字段的软删除。
